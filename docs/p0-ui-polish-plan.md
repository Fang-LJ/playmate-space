# P0 UI 对齐与体验收尾计划

本文档用于 P0 功能闭环完成后的 UI 对齐准备。当前阶段只整理 UI 对齐方案，不新增 P1 功能，不实现行程、投票、账本、AA、照片墙。

参考资料：

- Notion：`05 设计稿与截图记录`
- Figma：玩伴空间小程序 - 第一版低保真原型
- 本地截图：`docs/notion-screenshots`
- 当前小程序：`playmate-miniprogram/pages`

## 当前页面和 Figma 的主要差异

1. 活动列表页已经使用清新旅行风卡片，但还缺少 Figma 中的场景化标题、状态筛选和待处理提醒层级。
2. 创建活动页已有封面、表单和底部操作，但类型选择仍是 picker，不像 Figma 中的分段按钮；封面与时间区域也还没有形成完整的视觉分组。
3. 活动详情页已经有活动头图、信息卡和功能入口，但 Figma 里更强调“活动空间首页”：成员头像区、出发前入口、活动中/活动后入口、待处理提醒更清晰。
4. 活动邀请页已有分享加入链路页面，但还需要更贴近 Figma 的邀请卡片氛围，突出活动名称、创建者、成员数和加入按钮。
5. 登录页和我的页之前带有开发测试文案，本次已去除用户可见测试痕迹；后续需要进一步产品化视觉。
6. 成员列表和昵称编辑页是 P0 后补页面，目前可用但还没有高保真参照，后续只做轻量统一风格即可。

## 活动列表页如何对齐 Figma

目标：让首页更像“活动空间入口”，而不是普通列表。

建议调整：

- 顶部标题保留“我的活动”，增加场景化引导文案，例如“周末要去哪玩？”。
- 增加状态筛选标签：全部、规划中、进行中、已结束。
- 活动卡片保留白色卡片、大圆角、轻阴影。
- 卡片内保留封面图，默认封面使用浅绿、米白、浅黄的轻插画占位。
- 增加待处理提醒区域，例如“2 个投票待参与”“3 笔账单未结算”“有 12 张新照片”。P0 暂无投票/账本/照片真实数据时，不显示假数据；后续 P1-P4 接入后再启用。
- 状态标签使用低饱和色，不使用警告感强的橙红色。

暂不做：

- 不接入行程、投票、账本、照片真实摘要。
- 不新增 AI、小游戏、社区入口。

## 创建活动页如何对齐 Figma

目标：提升表单的轻量感和活动感。

建议调整：

- 将表单分为“基础信息”“活动类型”“封面与时间”“地点与描述”。
- 活动类型从 picker 改成按钮组或网格选项，选中态使用浅绿色背景和绿色边框。
- 封面上传区域与开始/结束日期形成同一视觉分组。
- 底部固定操作栏保持当前轻量方案：白色底栏、浅分割线、左侧取消、右侧创建活动。
- 输入框保持浅灰绿背景、轻边框、大圆角。

暂不做：

- 不做地图定位。
- 不做路线规划。
- 不做 AI 行程生成。

## 活动详情页如何对齐 Figma

目标：把详情页强化成“活动空间首页”。

建议调整：

- 顶部活动信息卡改为浅绿色背景或轻量封面卡，突出活动名称、时间、地点、状态。
- 成员区独立展示，使用头像圆点或头像列表，右侧提供“查看全部”。
- 功能入口分两组：
  - 出发前：成员、行程、投票。
  - 活动中 / 活动后：账本、AA 结算、照片墙。
- P0 当前只实现成员，行程/投票/账本/AA/照片墙入口可显示“暂未开放”，但不要写“后续接入”这类开发文案。
- 增加“待处理提醒”卡片位置；P0 可先隐藏或展示空状态，等对应模块接入后再展示真实提醒。
- 创建者操作区保持克制，不要抢占活动空间首页的主视觉。

暂不做：

- 不做行程、投票、账本、AA、照片墙真实功能。
- 不做回忆页。

## 活动邀请页如何对齐 Figma

目标：让分享进入时更像一张活动邀请卡，而不是普通详情页。

建议调整：

- 顶部使用活动封面或浅绿色邀请卡。
- 重点展示活动名称、时间、地点、创建者、成员数、活动状态。
- 主按钮固定为“加入活动”或“进入活动”，按钮使用绿色主色。
- 不可加入状态明确展示原因：活动已结束、活动已取消、你已被移除、链接失效。
- 未登录状态不展示技术原因，只提示“登录后即可加入活动”。
- 保留返回首页入口。

暂不做：

- 不展示完整成员列表。
- 不展示内部活动详情。

## 我的页 / 登录页如何产品化

目标：去除开发痕迹，保留正式用户入口。

已完成：

- 登录页已去掉 mockOpenid、接口路径、token 等开发文案。
- 我的页已去掉未登录处理验证、活动封面上传测试、接口路径说明。
- 我的页保留用户信息、我的活动入口、关于项目、退出登录。

后续轻量优化：

- 登录页插画区域可进一步做成清新旅行风视觉，不使用复杂插画。
- 我的页可增加“我创建的活动 / 我加入的活动”统计，但 P0 没有单独统计接口时先不做。
- 关于项目保持轻量，不做营销页。

## 哪些页面只需要轻量优化

- 成员列表页：统一卡片、头像、角色标签、移除按钮样式即可。
- 活动内昵称编辑页：保持简单表单，重点是输入体验和保存反馈。
- 链接失效页：保持低保真即可，突出原因和返回首页。
- 编辑活动页：复用创建活动页样式即可，不单独做复杂设计。

## 哪些页面暂时不做

以下页面或模块不在 P0 UI 对齐范围内：

- 行程列表页
- 新增 / 编辑行程页
- 投票列表页
- 创建投票页
- 投票详情页真实功能
- 账本列表页
- 新增 / 编辑消费页
- 消费详情页
- AA 结算页真实功能
- 照片墙页
- 照片详情页
- 回忆页
- AI 行程生成
- AI 文案
- 小游戏
- 微信支付
- OCR
- 地图路线规划
- Web 管理后台

## 建议执行顺序

1. 先统一登录页和我的页的正式用户文案。
2. 对齐活动列表页：标题、筛选、卡片层级。
3. 对齐创建活动页：类型选项、分区表单、底部操作栏。
4. 对齐活动详情页：活动空间首页结构、成员区、功能分组。
5. 对齐活动邀请页和链接失效页。
6. 轻量统一成员列表和昵称编辑页。
7. 微信开发者工具人工验收：模拟器、分享路径、图片上传、活动状态、成员移除。

## 实际已落地的 UI 改造内容

本轮已完成 P0 已实现页面的第一轮视觉精细化，范围仍限定在 P0，不新增 P1 功能。

### P0.5 第二轮体验收尾

- 继续补齐 `app.wxss` 主题 token：`--pm-warning`、`--pm-warning-soft`、`--pm-success`、`--pm-radius-button`、`--pm-shadow-soft`，并新增通用 `pm-hero-card`、`pm-button-like`、`pm-link`。
- 登录页强化品牌区和邀请感插画，保持“微信一键登录 / 暂不登录”的正式产品结构，开发环境提示继续弱化。
- 我的页移除“同步最新资料”主入口，资料卡改为可点击的编辑 / 登录入口，整体更像个人中心。
- 编辑资料页去除 P0.5 开发文案，头像区域改为更大的资料卡，联系电话说明改为正式产品说明。
- 活动列表页将角色枚举转为“我创建的 / 我加入的”，成员数和角色改为柔和标签展示。
- 活动详情页将“成员”入口标记为已开放，未开放模块统一弱化展示，创建者操作按钮改为统一按钮体系。
- 活动邀请页补充成员数和创建者摘要，顶部增加地点和日期层级，更像分享落地邀请页。
- 链接失效页统一到清新旅行风状态卡片。

### 全局样式

- 在 `app.wxss` 中补充统一 design tokens：浅绿背景、绿色主色、米黄点缀、文本色、边框色、危险色、圆角、阴影。
- 覆盖 TDesign MiniProgram 品牌色变量，主按钮、TabBar、加载等默认蓝色改为绿色系。
- 抽取公共样式：`pm-chip`、`pm-status`、`pm-form-card`、`pm-field`、`pm-input`、`pm-textarea`、`pm-bottom-bar`。

### 活动列表页

- 顶部改为更接近 Figma 的“我的活动 + 周末要去哪玩？”结构。
- 新增状态筛选 chip：全部、规划中、进行中、已结束。
- 活动卡片统一为白底大圆角、轻边框、弱阴影，保留封面、状态、地点、日期、成员数、角色。
- 默认封面使用浅绿 / 米黄轻插画占位，不再像普通灰色占位图。

### 创建活动页 / 编辑活动页

- 表单按 Figma 拆成基础信息、活动类型、封面与时间、地点与描述。
- 活动类型从 picker 改为 chip 按钮组。
- 封面上传区域改为轻量封面占位卡片。
- 日期选择改为卡片式开始 / 结束日期。
- 编辑页复用创建页样式，已结束活动仍只开放封面和描述编辑。

### 活动详情页

- 重点改为“活动空间首页”结构，而不是表格式详情。
- 顶部活动摘要卡使用浅绿色背景，突出活动名称、时间、地点、状态和描述。
- 增加成员摘要条和“查看全部”入口。
- 功能入口按 Figma 分组：
  - 出发前：成员、行程、投票
  - 活动中 / 活动后：账本、AA 结算、照片墙
- 未开放模块统一展示“暂未开放”，不使用开发口吻。
- 创建者操作区弱化为底部卡片，不抢占活动主视觉。

### 邀请页 / 链接失效页

- 邀请页改成独立邀请落地页，突出“朋友邀请你加入活动”、活动标题、状态、时间和地点。
- 活动信息卡保留地点、创建者、成员数、描述和不可加入原因。
- 链接失效页统一使用同一套浅绿风格和返回首页按钮。

### 登录页 / 我的页

- 登录页增加品牌标识和正式产品文案。
- 我的页改为个人信息卡 + 菜单入口，不再像调试页或设置页。
- 我的页入口保留：我的活动、关于玩伴空间、刷新用户信息、退出登录。

### 成员列表 / 昵称编辑页

- 成员列表统一头像、角色标签、“我”标签和危险移除按钮。
- 昵称编辑页复用公共表单和底部操作栏。

## 暂时无法完全还原的内容

- 首页活动卡片中的投票待参与、账单未结算、新照片等提醒，因为 P1-P4 模块尚未实现，当前不展示假数据。
- 活动详情页成员头像条目前只能用占位圆点表达氛围，完整头像需要成员接口进一步返回头像列表或详情页额外请求成员列表。
- 行程、投票、账本、AA、照片墙入口只做视觉入口和“暂未开放”，不接入真实功能。
- Figma 中的投票详情页和 AA 结算页不在当前 P0 可用功能范围内，本轮不落地。

## 建议截图验收清单

1. 活动列表页：无活动、已有活动、规划中 / 已结束 / 已取消状态卡片。
2. 创建活动页：无封面、上传封面后、类型 chip 选中、日期选择。
3. 活动详情页：规划中活动、已结束活动、已取消活动、创建者操作区、模块入口分组。
4. 活动邀请页：未加入可加入、已加入进入活动、已结束或已取消不可加入。
5. 链接失效页：缺少 code 或无效分享码。
6. 登录页：未登录进入、登录 loading、登录后跳转。
7. 我的页：未登录、已登录、有头像 / 无头像、退出登录。
8. 成员列表页：创建者视角、普通成员视角、移除按钮、当前用户“我”标签。
9. 活动内昵称编辑页：输入为空、保存成功、返回成员列表刷新。
